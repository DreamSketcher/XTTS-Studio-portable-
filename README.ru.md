## Что это

XTTS Studio запускает XTTS v2 локально и собирает вокруг него всё, что обычно приходится делать вручную:

- подготовку голосового референса;
- нормализацию и разбиение длинного текста;
- очередь задач и контроль качества чанков;
- библиотеку голосов и кэш эмбеддингов;
- RVC-постобработку;
- историю с wave-формой;
- экспорт WAV/MP3;
- опциональный облачный или локальный AI.

Основная генерация не отправляет голос и текст в облако. Сеть нужна только для функций, которые без неё невозможны: облачного AI, каталогов и загрузок моделей, проверки обновлений и установки некоторых компонентов.

| Задача | Интернет |
|--------|----------|
| XTTS с локальным референсом | не нужен |
| RVC с уже установленной моделью | не нужен |
| Локальная GGUF-модель после скачивания | не нужен |
| RVC-каталог, скачивание моделей, обновления | нужен |
| Groq / OpenRouter / другой облачный AI | нужен |

## Что можно делать

### 🎤 Клонировать голос

- референс примерно **10–20 секунд**;
- автоматический mono WAV 24 кГц, trim, нормализация и оценка SNR;
- библиотека `library/<voice>/`;
- повторное использование speaker embedding;
- длинные тексты обрабатываются по чанкам и склеиваются в один файл.

### 🎭 Менять тембр через RVC

- локальные `.pth` и optional `.index`;
- каталоги **★ Подборка · 🆕 Новые · 🔥 Топ** и live-поиск;
- demo модели до скачивания;
- preview уже скачанной модели;
- отдельный parameter preview на первых 6 секундах вашего voice reference;
- настройка Index, Pitch shift и f0 method;
- очистка preview и остатков прерванных загрузок.

> Index имеет смысл только при наличии подходящего `.index`. RVC меняет тембр готового аудио, но не исправляет плохую дикцию базового XTTS.

### 📚 Озвучивать длинные материалы

- главы книг, сценарии, лекции, закадр;
- четыре пресета: **Высокое качество · Нарратив · Динамика · Экспрессия**;
- QC с повторной генерацией подозрительных чанков;
- умные паузы, просодия, trim и de-esser;
- пакетная обработка нескольких TXT-файлов;
- последовательная очередь с отменой задач.

### 📝 Работать с текстом

- числа и даты → слова;
- словарь произношений с пользовательским приоритетом;
- ёфикация;
- защита инициалов вроде `А. С. Пушкин`;
- автоматическое разбиение примерно по 50–175 символов;
- RU/EN текст и адаптивное определение языка.

### 🤖 Подключать AI только когда он нужен

- AI Conductor назначает параметры и паузы по чанкам;
- optional rewrite стиля;
- AI-чат с несколькими провайдерами и fallback-цепочкой;
- Groq, OpenRouter, custom OpenAI-compatible API;
- локальные GGUF-модели через `llama-cpp-python`.

### 🎧 Возвращаться к результатам

В истории хранятся последние 100 генераций. На каждой карточке есть wave-форма: можно проиграть, остановить, перейти кликом к нужному месту и вернуть текст в редактор кнопкой **↩**.

## Реальные сценарии

- **Аудиокнига:** чистый спокойный референс → пресет «Нарратив» → глава целиком → проверка спорных мест в истории.
- **Голос персонажа:** сначала разборчивый XTTS без RVC → затем `.pth` → parameter preview → подбор Pitch/Index/f0.
- **Набор реплик:** несколько TXT через batch → очередь → отдельные WAV/MP3.
- **Приватный текст:** AI выключен или выбран локальный GGUF; синтез и RVC остаются на компьютере.

## Установка

1. Скачайте portable-архив в [GitHub Releases](https://github.com/DreamSketcher/XTTS-Studio/releases).
2. Запустите `XTTS Studio.exe`.
3. Выберите референс и короткую тестовую фразу.
4. Только после проверки запускайте главу или batch.


Сборка сразу работает на CPU. Для совместимой NVIDIA GPU установите CUDA-вариант через **⚙ Настройки → Ускорение**.

> Portable-архив занимает несколько гигабайт, потому что включает Python-окружение и модели. Не устанавливайте зависимости поверх системного Python.

---

## Что такое `XTTS Studio.exe`

`XTTS Studio.exe` — небольшой лаунчер, преобразованный из стартового BAT-файла. В нём нет XTTS-модели, пользовательских настроек или исходного кода приложения. Он хранит только пути запуска bundled Python, путь к `gui.py` и иконку.

Лаунчер включён в систему обновлений. Если структура portable-сборки изменится и пути к runtime или окружению придётся переписать, updater заменит `.exe` вместе с остальными изменёнными файлами.

---

## Как обновляется приложение

Полный архив нужен для первой установки или редкого несовместимого обновления. Обычные релизы устанавливаются встроенным клиентским updater без повторного скачивания всей тяжёлой сборки.

```text
проверка версии
  → список изменённых файлов
  → скачивание во staging
  → SHA256
  → backup
  → замена файлов
  → подтверждение успешного запуска
```

Если новый запуск не подтвердился, updater может вернуть backup. Полная переустановка требуется, когда текущая версия ниже `min_app_version` или структура portable-сборки стала несовместимой.

## Требования

| | Минимально | Рекомендуется |
|---|-----------|--------------|
| ОС | Windows 10/11 x64 | Windows 10/11 x64 |
| RAM | 8 ГБ | 16+ ГБ для длинных задач и local LLM |
| CPU | современный x64 | больше ядер сокращают ожидание |
| GPU | не требуется | NVIDIA 4+ ГБ VRAM, CC 6.0+ |
| Диск | portable-сборка + модели | запас под RVC, GGUF, outputs и кэши |

CPU работает сразу, но XTTS, RVC и local LLM могут быть заметно медленнее real-time. CUDA ускоряет поддерживаемые операции, но требует совместимой NVIDIA GPU и правильного PyTorch.
