## 🚀 О проекте

**XTTS Studio** — полностью офлайн инструмент для синтеза речи и клонирования голоса.

## ✨ Возможности

### 🎤 Синтез и клонирование
- Полностью офлайн — никаких внешних запросов
- Portable — одна папка, любой Windows ПК
- Клонирование голоса по референсу 10–20 секунд
- Библиотека голосов с кэшем Speaker Embedding
- Поддержка длинных текстов без ограничений
- Авто-переключение языка RU/EN внутри одного текста
- Автодетект CUDA/CPU с универсальным набором зависимостей — если видеокарта не поддерживается, приложение само переключается на CPU без ошибок

### 🖥 Интерфейс
- 🌗 **Темы:** тёмная и мягкая светлая («молочная») из коробки, плюс встроенный **конструктор тем** — можно собрать свою цветовую схему, а не только переключать готовые
- 🌐 **Два языка интерфейса:** русский и английский (RU/EN), включая AI-чат и настройки провайдеров
- 📐 Адаптивный тулбар — панели растягиваются под размер окна
- ◀ Выдвижная левая панель с запоминанием положения
- 🔠 Регулируемый размер текста в окне ввода (ползунок «Aa»)
- Все настройки вида сохраняются между сессиями
- 🔄 **Автообновление** — поэтапная (staged) загрузка новой версии, проверка целостности через **SHA256**, автоматический бэкап и откат (rollback) при сбое, определение необходимости полной переустановки по `min_app_version`

### 🧠 Обработка текста
- Числа → слова автоматически
- Аббревиатуры → фонетический словарь (авто + ручной)
- Смысловые и пунктуационные паузы — автоматически
- Нормализация текста перед генерацией

### 🎛 Управление качеством
- 4 пресета: ⭐ Высокое качество / 📖 Нарратив / ⚡ Динамика / 🎭 Экспрессия
- Тонкая настройка: temperature, top_p, repetition_penalty, скорость, trim
- Контроль качества чанков — авто-перегенерация при повторах и обрывах
- Де-эссер, RMS-нормализация громкости, авто-обрезка тишины
- Кэш чанков — повторная генерация того же текста не тратит время

### 🤖 AI-модуль (опционально)
- **AI Conductor** — анализирует текст и назначает параметры XTTS для каждого чанка индивидуально
  - Уровень 1: temperature, speed, паузы по контексту и интонации
  - Уровень 2: rewrite текста под заданный жанр или настроение (с negative prompt)
  - Оба уровня полностью независимы — включение/выключение одного не влияет на другой (проверка задублирована и в логике кондуктора, и в раннере)
- **AI чат** — встроенный чат-ассистент с историей сессий и поиском
  - Режим редактора текста и режим свободного чата
  - Кнопка «Улучшить» — технический rewrite текста для лучшего TTS
  - Раскрывающиеся карточки настроек провайдера (аккордеон) вместо длинной формы
- **Окно «AI статус»** — цепочка провайдеров и статус каждого в реальном времени
- Поддержка цепочки провайдеров: Groq, OpenRouter, локальные LLM (офлайн, без ключей), кастомные OpenAI-совместимые
- Каталог провайдеров, библиотека ключей; выбранный пользователем провайдер всегда идёт первым в цепочке
- **Локальные LLM (офлайн, без ключей и интернета)**
  - Автоматическое сканирование окружения ПК (наличие GPU, CUDA, объём VRAM/RAM) для подбора совместимой конфигурации
  - Автоматическая установка необходимых библиотек под выбранную модель — без ручной возни с pip
  - Встроенный каталог поддерживаемых локальных моделей с загрузкой прямо из интерфейса
  - Возможность подключить собственную модель — достаточно указать папку с её файлами

### 📋 Прочее
- Очередь задач с отменой
- Пакетная обработка TXT-файлов
- История генераций с возвратом текста
- Подсветка текущего чанка в реальном времени
- Статистика: время, чанки, голос, скорость
- Автосохранение настроек между сессиями
- Экспорт в WAV и MP3

## 🚀 Быстрый старт

1. Скачайте и распакуйте архив
2. Не используйте путь с кириллицей
3. Запустите `XTTS Studio.exe`
4. Выберите или загрузите голосовой референс
5. Введите текст
6. Нажмите **🚀 ГЕНЕРИРОВАТЬ**
7. Результат сохраняется в `outputs/` — кнопка **🎵 Аудио**

## ⚙️ Как работает

```
Референс → авто-обработка → библиотека голосов
   ↓
Текст → нормализация → числа в слова → аббревиатуры
   ↓
(опционально) AI-улучшение / rewrite текста
   ↓
Разбивка на чанки → простановка пауз
   ↓
(опционально) AI Conductor — параметры для каждого чанка
   ↓
Генерация с контролем качества и кэшированием
   ↓
Сборка → нормализация громкости → де-эссер → WAV / MP3
```

## 🧠 Словарь произношений

Примеры встроенных правил:

```
AI      → эй ай
CPU     → си-пи-ю
GPU     → джи-пи-ю
OpenAI  → ОпенЭйАй
```

Словарь пополняется автоматически при генерации, а если включён **AI Conductor** — он дополнительно помогает словарю: анализирует транслит и сам добавляет исправления в категорию `ai_corrected`, то есть активный кондуктор фактически «доучивает» словарь параллельно с озвучкой.
Редактирование — кнопка **📖 Словарь** (добавление, изменение, удаление правил).

- **Категории правил с приоритетом:** `builtin → auto → ai_corrected → custom` — более специфичное правило перекрывает более общее
- **Diagnostic dry-run режим** — предпросмотр того, что изменится, без записи в файл
- **Автоматические таймстамп-бэкапы** в `word_rules_backups/` перед каждым сохранением
- Метаданные у каждой записи (категория, источник)

## 🔬 Модули engine/ — по зонам ответственности

### Пайплайн генерации
- **`tts_runner.py`** — тонкая точка входа; основная логика `run_tts()` перенесена в пакет **`engine/tts/`**: normalize → word replacer → chunk → conductor → generate → merge. Ленивая загрузка модели (thread-safe singleton), кэш embedding и готовых чанков (md5), авто-trim по тишине, RMS-нормализация громкости.
- **`engine/tts/qc.py`** — контроль качества: детектор зацикливания + валидатор длительности, авто-перегенерация.
- **`engine/tts/device.py`** — автодетект CUDA/CPU с фолбэком на CPU, если видеокарта не поддерживается.
- **`engine/tts/cache.py`** / **`export.py`** — кэш чанков и экспорт в WAV/MP3.
- **`chunker.py`** — разбивка на предложения, нарезка длинных, объединение коротких, проверка плохого начала/конца чанка.
- **`normalizer.py`** — числа→слова, аббревиатуры, пунктуация; отдельно ритм для латинских/кириллических аббревиатур и CamelCase.
- **`word_replacer.py`** — фонетические замены по словарю с приоритетом категорий (`builtin → auto → ai_corrected → custom`), автотранслитерация терминов; **`word_replacer_window.py`** добавляет diagnostic dry-run режим и автоматические таймстамп-бэкапы в `word_rules_backups/` перед каждым сохранением.
- **`smart_pauses.py` / `prosody_layer.py`** — паузы и смысловая просодия; **оба пропускаются, если активен AI Conductor** — тогда паузы и temperature schedule берутся из `conductor_map`.

### AI-модуль
- **`ai_conductor.py`** — `conduct()`: один вызов на весь текст, анализирует чанки и возвращает параметры озвучки (temperature/top_p/repetition_penalty/speed/pause_after_ms) для каждого. Опционально — переработка текста под стиль (`rewrite_enabled`) и проверка транслита (`corrections` → `word_rules.json`). При ошибке ИИ — фолбэк на дефолтные параметры, генерация не прерывается. Уровни 1 и 2 явно разделены флагами и в этом модуле, и в `tts_runner.py`, чтобы они не могли случайно повлиять друг на друга.
- **`gui/chat_window.py` + `gui/chat_window/`** — окно AI-чата: `chat_window.py` — главный файл интерфейса (сборка UI), а пакет `chat_window/` содержит его подмодули: рендер сообщений, ввод, история и поиск сессий, экспорт переписки, режим редактора текста и режим свободного чата, аккордеон настроек провайдеров, горячие клавиши, индикатор набора текста. Логика генерации ответа и управление сессиями вынесены во вложенный `chat_window/engine/`. Полностью локализовано (RU/EN).
- **`gpt_client.py`** — цепочка облачных провайдеров (активный → встроенные → кастомные), управление ключами и моделями, каталог провайдеров.
- **`local_llm_client.py`** / **`local_env_section.py`** / **`env_setup.py`** — поддержка локальных (офлайн) LLM как альтернативы облачным провайдерам: автоматическое сканирование окружения ПК (GPU/CUDA/VRAM), автоматическая установка нужных библиотек под выбранную модель, встроенный каталог поддерживаемых моделей и возможность подключить свою модель из папки.
- **`gui/ai_status_window.py`** — диагностическое окно, показывающее цепочку провайдеров и статус каждого.

### Голос и аудио
- **`reference_processor.py`** — конвертация референса в WAV, SNR-проверка, кэш.
- **`voice_manager.py`** — сканирование `library/`, список голосов, активный голос.
- **`de_esser.py`** — подавление шипящих на финальном файле.

### Инфраструктура
- **`task_manager.py` / `task_models.py`** — многопоточная очередь генерации с отменой по id.
- **`updater.py`** — система обновлений: поэтапная (staged) загрузка новой версии, проверка целостности каждого файла через **SHA256**, автоматический бэкап текущей версии и rollback при сбое, определение необходимости полной переустановки через `min_app_version` (корректная обработка пробелов в именах файлов через URL-кодирование), самоперезапуск.
- **`i18n.py`** — словарь переводов RU/EN (350+ ключей), автозагрузка сохранённого языка.

## 🔐 Как устроено обновление

Система обновлений построена по принципу «безопасно, даже если что-то пойдёт не так»:

1. **Проверка версии** — текущая версия сверяется с `version.json`; поле `min_app_version` определяет, достаточно ли обычного патча или требуется полная переустановка.
2. **Поэтапная (staged) загрузка** — новая версия скачивается во временную папку, не затрагивая рабочие файлы приложения.
3. **Проверка SHA256** — каждый скачанный файл сверяется по контрольной сумме; при несовпадении хэша обновление прерывается и рабочая версия остаётся нетронутой.
4. **Бэкап и откат (rollback)** — перед заменой файлов создаётся резервная копия текущей версии; если применение обновления завершилось ошибкой, приложение автоматически откатывается на бэкап.
5. **Самоперезапуск** — после успешного применения обновления приложение перезапускается само.

Если автообновление всё же не сработало, `XTTS_DIAG.bat` предоставляет форсированный вариант восстановления/переустановки.

Приложение разработано с использованием AI-инструментов: **Claude**, **ChatGPT** и других.
