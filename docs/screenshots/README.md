# Screenshots — docs/screenshots/

Каталог для скриншотов UI (TASK-013). Минимум 5 PNG, общий вес <2 MB.

## Какие нужны скриншоты (привязка к разделам DOCUMENTATION)

| Файл | Что показывать | Раздел docs |
|------|----------------|-------------|
| `01_main_generation.png` | Главное окно + процесс генерации (выбранный референс, текст, статус-бар) | Быстрый старт / Как проходит генерация |
| `02_rvc_browser.png` | RVC-браузер с параметрическим preview (▶ sample, dropdown, Index/Pitch/f0) | RVC: выбор модели, preview и кэш |
| `03_history_waveform.png` | Окно истории с wave-формой, ▶/■, перемоткой по клику | Очередь, история и пакетная обработка |
| `04_quality_settings.png` | Окно настроек качества (вкладки RVC/Обрезка/Вывод/XTTS, ползунки) | Пресеты и параметры качества |
| `05_startup_recovery.png` | Startup recovery / окно диагностики (critical/optional, recovery) | Системное окружение и восстановление |

## Требования

- Общий вес <2 MB: PNG с оптимизацией/сжатием; при необходимости — уменьшить разрешение.
- Содержимое без приватных данных: убрать реальные пути, API-ключи, личный текст.
- Имена файлов фиксированы (см. таблицу) — на них ссылается DOCUMENTATION.*.md.

## Встраивание

Шаблоны уже есть в обоих DOCUMENTATION.*.md (ищите `docs/screenshots/`). После добавления
PNG проверьте, что изображения подгружаются и общий вес в пределах лимита.
