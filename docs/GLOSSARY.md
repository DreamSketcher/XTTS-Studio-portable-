# Глоссарий

Единая терминология XTTS Studio AI. Используется во всей документации и UI.

| Термин | Определение |
|--------|-------------|
| **XTTS** | Модель текст-в-речь, на которой построен проект. Везде, где не уточнено, имеется в виду **XTTS v2** — единственная используемая модель. Формулировки «XTTS» и «XTTS v2» взаимозаменяемы и означают одно и то же. |
| **XTTS v2** | Конкретная версия модели XTTS (Coqui Public Model License). Базовый движок синтеза речи проекта. |
| **XTTS Studio** / **XTTS Studio AI** | Само приложение (портативное Windows-окружение + UI), построенное поверх XTTS v2. |
| **RVC** | Retrieval-based Voice Conversion — второй этап обработки: меняет тембр готового XTTS-аудио. RVC-модель — файл `.pth` (+ опционально `.index`). |
| **RVC-модель** | Голосовая модель для RVC (`.pth`). Может быть собственной или community (из каталога). |
| **GGUF-модель** / **локальная GGUF** | Локальная LLM в формате `.gguf` для AI-функций (чат, AI Conductor, улучшение текста). Запускается в процессе через `llama-cpp-python`. Оба варианта названия означают одно и то же. |
| **референс** / **voice reference** | Короткий (10–20 с) образец голоса, по которому XTTS клонирует тембр и манеру. |
| **чанк** / **chunk** | Фрагмент текста, который XTTS генерирует отдельно (≈50–175 символов). Склеивается с паузами в финальное аудио. Оба термина означают одно и то же. |
| **QC** | Quality Control — контроль качества чанков: авто-перегенерация бракованных фрагментов (повторы, неправдоподобная длительность), до 3 попыток. |
| **пресет** | Набор значений параметров генерации (temperature, top_p, top_k, repetition penalty, speed, prosody) под задачу: «Высокое качество», «Нарратив», «Динамика», «Экспрессия». |
| **AI Conductor** | AI-режим, который анализирует весь текст одним вызовом и назначает параметры XTTS для каждого чанка индивидуально (заменяя prosody/smart pauses). |
| **catalogue** / **каталог** | Встроенный индекс моделей (seed + кэш + live voice-models.com для RVC; встроенный GGUF-каталог). Наличие в каталоге не означает разрешения на коммерческое использование. |
| **DPAPI** | Windows Data Protection API — используется для защиты API-ключей в `gpt_settings.json`. |
| **Ed25519** | Алгоритм подписи update-манифеста (`json/version.json.sig`); реальный gate целостности релиза до применения code-signing сертификата. |
| **bundled Python** | Собственный Python-окружение приложения (`python\runtime`, `python\xtts_env`), не системный. |
| **CPML** | Coqui Public Model License — лицензия модели XTTS v2; ограничивает коммерческое использование. |
